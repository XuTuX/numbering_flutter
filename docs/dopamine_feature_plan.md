# NUMBERING 도파민 강화 기능 기획안

## 1. 문서 목적

이 문서는 NUMBERING의 핵심 퍼즐 규칙을 바탕으로 다음 세 가지 기능의 의도와 동작을 정리한 사전 기획안이다.

1. 실시간 라이벌 추월
2. 최대 길이 매칭으로 발동하는 허니 러시
3. 4초 콤보 유지 시간 표시

현재 단계에서는 기능 방향만 정리한다. 이 문서를 작성하는 과정에서 게임 코드, 점수 공식, 데이터베이스 또는 기존 핵심 규칙은 변경하지 않는다.

---

## 2. 전체 경험 방향

세 기능은 각각 별개로 보이지만 하나의 반복 흐름을 만든다.

> 가까운 라이벌을 확인한다 → 콤보 제한 시간을 보며 빠르게 연결한다 → 10칸 연결로 허니 러시를 발동한다 → 라이벌을 추월한다

핵심은 보상 종류를 많이 늘리는 것이 아니라 한 판 안에 다음 두 가지 감정을 만드는 것이다.

- **추격:** 조금만 더 점수를 얻으면 누군가를 추월할 수 있다.
- **폭발:** 어려운 10칸 연결에 성공하면 4초 동안 평소와 완전히 다른 플레이가 열린다.

---

## 3. 기능 1 — 실시간 라이벌 추월

### 3.1 목적

전체 1위처럼 멀리 있는 목표 대신, 현재 플레이어 바로 위에 있는 한 명을 보여준다. 플레이 도중 점수가 라이벌의 기록을 넘어서는 순간을 즉시 축하해 경쟁의 재미를 강화한다.

여기서 `실시간`은 게임 중 서버 순위를 계속 갱신한다는 의미가 아니다. **게임 시작 전에 불러온 랭킹 스냅샷을 기준으로 현재 점수의 추월 여부를 즉시 반영한다**는 의미다. 이렇게 하면 네트워크 지연이나 플레이 중 순위 변동으로 인한 혼란을 줄일 수 있다.

### 3.2 기본 흐름

1. 게임 시작 전에 플레이어 주변 랭킹과 점수를 불러온다.
2. HUD에 현재 추월 대상 한 명을 작은 닉네임 스티커로 표시한다.
3. 플레이어 점수가 목표 점수를 넘으면 추월 연출을 재생한다.
4. 기존 라이벌 스티커는 화면 밖으로 밀려난다.
5. 다음 순위의 라이벌이 새로운 목표로 들어온다.

표시 예시:

```text
14위 · 벌꿀왕
3,820점까지 240점
```

추월 순간 예시:

```text
14위 추월!
다음 상대: 13위 꿀벌이
```

### 3.3 목표 점수 판정

- 동점은 추월로 처리하지 않는다.
- UI에는 `라이벌 점수 + 1점`을 실제 추월 목표로 사용한다.
- 한 판에서 여러 명을 넘을 수 있다.

### 3.4 모드별 기준

- **일반 모드:** 현재 주간 랭킹의 내 주변 플레이어를 사용한다.
- **오늘의 퍼즐 공식 도전:** 해당 날짜의 일일 랭킹 스냅샷을 사용한다.
- **연습·튜토리얼·리플레이:** 라이벌 HUD를 표시하지 않는다.
- **비로그인 또는 랭킹 로딩 실패:** 기능을 숨기고 게임 진행에는 영향을 주지 않는다.

### 3.5 화면 연출

- 기본 상태에서는 보드를 가리지 않는 작은 스티커 형태로 표시한다.
- 목표까지 남은 점수는 매 점수 변화마다 숫자만 갱신한다.
- 추월 시 0.8~1.0초 이내의 짧은 슬라이드·바운스 연출을 사용한다.
- 게임 플레이를 멈추거나 입력을 차단하지 않는다.
- 한 번에 여러 명을 넘었을 경우 모든 라이벌을 차례대로 오래 보여주지 않고 `3명 추월!`처럼 합쳐 표현한다.

### 3.6 주의사항

- 게임 중 서버를 반복 호출하지 않는다.
- 시작 시점 스냅샷과 최종 서버 순위가 다를 수 있으므로 게임 중에는 `현재 14위 확정`보다 `14위 기록 추월`처럼 표현한다.
- 최종 순위는 점수 제출이 완료된 뒤 결과 화면 또는 랭킹 화면에서 확정한다.

## 4. 기능 2 — 최대 길이 허니 러시

### 4.1 핵심 콘셉트

기존의 매칭 누적 게이지 방식은 사용하지 않는다. 현재 컬러 바의 최대 길이인 **10칸 전체를 한 번의 정상적인 경로로 연결했을 때** 즉시 특별 보상 구간을 연다.

> 10칸 정상 매칭 성공 → 기존 보드에 꿀빛 테두리가 자연스럽게 이어짐 → 4초 동안 자유 드래그로 점수 획득 → 원래 퍼즐로 복귀

어려운 플레이를 성공한 직후 규칙이 잠시 단순해지고, 플레이어가 생각보다 손을 빠르게 움직이며 보상을 쓸어 담는 순간을 만드는 것이 목적이다.

### 4.2 확정 방향

- 발동 조건은 유효한 **10칸 경로 매칭 성공**이다.
- 허니 러시 지속 시간은 **4초**다.
- 발동 중에도 타일의 원래 색은 유지하고 앰버 테두리와 은은한 중심 광원만 더한다.
- 발동 중에는 기존 컬러 바의 색 순서를 맞추지 않아도 드래그로 점수를 얻을 수 있다.
- 허니 러시에서 얻는 점수에는 **시간 회복이 없다.**
- 기존의 매칭 누적형 꿀 게이지는 만들지 않는다.

### 4.3 권장 세부 규칙 초안

아래 내용은 자유 드래그의 악용과 점수 폭주를 막으면서 4초의 속도감을 살리기 위한 권장안이다. 구현 전에 최종 확정이 필요하다.

1. 서로 인접한 타일만 연속해서 드래그할 수 있다.
2. 한 경로 안에서 같은 타일을 중복 방문할 수 없다.
3. 손을 떼면 지나간 허니 러시 타일 수에 따라 즉시 점수를 얻는다.
4. 한 번 점수를 준 타일은 빠르게 축소·페이드되고, 해당 허니 러시 동안 다시 점수를 주지 않는다.
5. 플레이어는 4초 동안 손을 여러 번 떼고 다시 드래그할 수 있다.
6. 허니 러시 중에는 보드 타일과 컬러 바의 실제 데이터가 바뀌지 않는다.
7. 4초가 끝나면 꿀빛 테두리와 수집 상태가 서서히 사라지며 기존 보드로 돌아온다.

이 방식이면 아무 방향으로 빠르게 드래그할 수 있지만, 한 타일을 반복 문질러 무한히 점수를 얻는 문제는 발생하지 않는다. 보드 전체 42칸이 한 번의 허니 러시에서 얻을 수 있는 자연스러운 최대 수집량이 된다.

### 4.4 점수 방식

점수 수치는 밸런스 테스트 후 결정한다. 우선 아래 두 방식 중 하나를 선택할 수 있다.

#### 안 A — 타일당 고정 점수

```text
획득 점수 = 수집한 허니 러시 타일 수 × 타일당 점수
```

- 이해하기 쉽고 손을 빠르게 움직인 만큼 보상받는다.
- 권장 시작값은 타일당 20~30점 범위다.

#### 안 B — 한 번에 연결한 길이에 따른 가속 보너스

```text
짧은 드래그보다 긴 드래그의 타일당 가치가 조금 더 높아진다.
```

- 기존 게임의 긴 경로 정체성을 유지할 수 있다.
- 4초 동안 계산을 이해하기 어려울 수 있으므로 점수 팝업을 명확히 보여줘야 한다.

초기 구현에는 단순하고 조정하기 쉬운 **안 A**가 적합하다.

### 4.5 시간·콤보·컬러 바 관계

- 일반 제한 시간은 허니 러시 4초 동안에도 계속 감소하는 방향을 기본안으로 한다.
- 허니 러시 자유 드래그는 시간 보너스를 발생시키지 않는다.
- 허니 러시 자유 드래그는 기존 컬러 바를 소비하지 않는다.
- 허니 러시 자유 드래그는 일반 매치 수, 최장 경로, 일반 콤보에 포함하지 않는 것을 권장한다.
- 발동 원인이 된 10칸 정상 매칭은 기존 점수·시간 보너스·콤보 규칙을 그대로 적용하는 것을 기본안으로 한다.
- 허니 러시 4초 동안에도 기존 콤보 제한 시간은 계속 흐른다.

### 4.6 시각·사운드 연출

#### 진입

- 10칸 경로가 순서대로 빛난 뒤 기존 타일 테두리로 앰버 광원이 이어진다.
- `HONEY RUSH!` 문구를 짧게 표시한다.
- BGM 위에 4초짜리 고조 레이어 또는 별도 효과음을 재생한다.
- 10칸 성공 판정 즉시 0.1초 안에 전용 패널과 앰버 테두리가 나타난다.
- 허니 러시 4초 입력과 기존 매치 제거·보드 리필 애니메이션은 동시에 진행한다.

#### 진행

- 컬러 바를 유지한 노란 패널 상단에 `4.0 → 0.0` 타이머를 표시한다.
- 드래그해 수집한 타일은 빠르게 축소·페이드되어 사라진다.
- 손가락 경로를 따라 꿀방울·별·작은 벌 파티클을 표시한다.
- 점수는 드래그를 끝낸 위치에서 즉시 떠오른다.

#### 종료

- `TIME!` 같은 실패 표현 대신 짧은 종소리와 함께 자연스럽게 원래 색으로 복귀한다.
- 총 허니 러시 획득 점수를 `HONEY +720`처럼 한 번 요약한다.
- 즉시 일반 입력을 다시 허용한다.

### 4.7 공식 랭킹 적용 시 고려사항

허니 러시가 실제 점수를 발생시키면 리플레이와 서버 검증에도 다음 정보가 포함되어야 한다.

- 10칸 정상 매칭에 의한 정확한 발동 여부
- 허니 러시 시작·종료 시각
- 각 자유 드래그의 경로와 입력 시각
- 같은 타일을 중복 수집하지 않았는지 여부
- 허니 러시 점수와 일반 점수의 분리 기록

공식 도전에 넣으려면 모든 참가자에게 같은 규칙을 적용하고 서버에서 검증할 수 있어야 한다. 검증이 준비되기 전에는 일반 모드에서 먼저 운영하는 것이 안전하다.

## 5. 기능 3 — 4초 콤보 유지 시간 표시

### 5.1 목적

현재 콤보는 이전 성공 후 4초 이내에 다음 매칭을 성공해야 이어지지만, 플레이어가 남은 시간을 정확히 알기 어렵다. 이를 컬러 바 상단의 작은 원형 타이머로 보여줘 다음 경로를 빠르게 찾게 한다.

### 5.2 기본 동작

- 정상 매칭 성공 직후 4초 타이머가 시작된다.
- 다음 정상 매칭에 성공하면 타이머가 다시 4초로 채워진다.
- 시간이 끝나면 링이 사라지고 다음 성공은 새 콤보 1로 시작한다.
- 재시작·나가기 확인창과 허니 러시 중에도 타이머 표시와 실제 콤보 시간은 계속 흐른다.
- 튜토리얼과 리플레이에서는 필요에 따라 숨긴다.

### 5.3 권장 형태

컬러 바 상단의 `COMBO` 문구 옆에 작은 원형 링을 사용한다.

- 4.0~2.0초: 앰버
- 2.0~0.8초: 진한 오렌지
- 0.8~0.0초: 코랄색 점멸
- 성공 시 링이 즉시 다시 차오르며 가벼운 펄스
- 숫자 카운트다운은 화면을 복잡하게 만들 수 있으므로 기본적으로 표시하지 않고 색과 감소량으로 전달한다.

원 안에는 현재 콤보 수를 표시해 남은 시간과 콤보 상태를 한곳에서 읽을 수 있게 한다.

### 5.4 콤보 종료 표현

- 콤보가 끝났다는 큰 실패 문구는 표시하지 않는다.
- 링이 부드럽게 사라지고 작은 효과음만 재생한다.
- 높은 콤보가 끊겼을 때도 점수 차감이나 강한 부정적 햅틱은 사용하지 않는다.

### 5.5 허니 러시와의 관계

허니 러시 자유 드래그는 일반 콤보를 올리지 않는다. 허니 러시 진입 시 컬러 바 패널이 짧은 페이드로 전용 타이머로 전환되고, 종료 후 기존 콤보 링으로 돌아온다.

콤보 타이머는 허니 러시 동안에도 중단 없이 계속 감소한다.

## 6. 기능 간 연결

### 6.1 인게임 HUD 우선순위

동시에 모든 정보를 강조하면 보드 집중도가 떨어진다. 상태별 우선순위는 다음과 같다.

1. 허니 러시 진행 중: 허니 러시 타이머와 보드의 앰버 테두리가 최우선
2. 일반 플레이 중: 콤보 링이 최우선
3. 라이벌 정보: 항상 작게 유지하고 추월 순간에만 잠깐 강조
4. 시간 임박 경고: 남은 시간이 5초 이하라면 콤보 링보다 제한 시간 경고가 우선

### 6.2 한 판의 이상적인 감정 흐름

```text
라이벌 확인
    ↓
일반 매칭과 콤보 추격
    ↓
10칸 경로 발견
    ↓
HONEY RUSH 4초
    ↓
라이벌 기록 추월
    ↓
게임 종료와 기록 확정
```

## 7. 구현 시 확정한 항목

초기 구현에서는 다음 값으로 확정했다.

1. 허니 러시 점수: 타일당 25점
2. 허니 러시 최소 드래그 길이: 2칸
3. 발동 원인인 10칸 정상 매칭: 기존 시간 보너스 유지
4. 적용 모드: 일반 모드 우선 적용, 공식 일일 도전 제외
5. 라이벌 스냅샷: 최대 20명
6. 콤보 시간 표시: 컬러 바 상단의 작은 원형 링

## 8. 권장 도입 순서

### 1단계 — 정보와 동기 강화

- 콤보 유지 시간 표시
- 실시간 라이벌 추월

기존 점수 규칙을 크게 바꾸지 않으면서 효과를 확인할 수 있다.

### 2단계 — 허니 러시 일반 모드 적용

- 10칸 발동 조건
- 4초간 원래 색을 유지하는 앰버 테두리 보드
- 자유 드래그 점수
- 시간 회복 없음
- 허니 러시 전용 기록 및 리플레이 데이터

### 3단계 — 공식 모드와 서버 검증

- 허니 러시 입력 검증
- 공식 일일 도전 적용 여부 결정
- 점수 밸런스 조정

## 9. 확인할 지표

기능 적용 후에는 단순 플레이 횟수뿐 아니라 다음 지표를 확인한다.

- 10칸 경로 성공률과 한 판당 허니 러시 발동 횟수
- 허니 러시 1회당 평균 수집 타일과 획득 점수
- 라이벌 HUD를 본 사용자의 추월 성공률
- 추월 직후 플레이 지속 시간
- 결과 화면 노출 후 `다시 시작하기` 선택률
- 콤보 링 도입 전후 평균·최대 콤보 변화
- 허니 러시가 전체 점수에서 차지하는 비율

허니 러시가 너무 자주 발동하면 희소성이 사라지고, 너무 드물면 존재를 경험하기 어렵다. 10칸 경로의 실제 성공률을 측정한 뒤 발동 조건은 유지하되 튜토리얼·연출·경로 인지 보조를 조정하는 방향이 적합하다.
