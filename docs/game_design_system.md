# NUMBERING 게임 UI 적용 규칙

이 문서는 [`design_system.md`](design_system.md)의 게임 화면 보충 규칙이다. 색상, 카드, 타이포그래피, 그림자 규칙은 공통 디자인 시스템이 우선한다.

## 1. 게임 화면의 목표

게임 플레이에서는 장식보다 퍼즐 상태를 빠르게 읽는 것이 중요하다. 화면 외곽 UI는 조용하게 유지하고, 사용자의 시선은 문제·수식·남은 시간·결과에 집중시킨다.

## 2. 공통 시스템에서 허용되는 예외

- 퍼즐 블록은 서로 구분하기 위해 고채도 기능 색상을 사용할 수 있다.
- 성공, 실패, 콤보, 시간 임박 피드백은 색상과 짧은 모션을 함께 사용할 수 있다.
- 드래그 또는 선택 중인 블록은 가벼운 `AppShadows.tileShadow`를 사용할 수 있다.
- 결과 화면의 최고 기록과 점수는 일반 화면보다 큰 숫자 계층을 사용할 수 있다.

예외는 상태 전달에만 사용한다. 게임 보드 주변 카드, 상단 바, 다이얼로그는 공통 미니멀 규칙을 그대로 따른다.

## 3. 게임 HUD

- 현재 문제와 입력 영역을 가장 크게 배치한다.
- 숫자 영역 위에는 `18–32dp`의 여백을 확보하고, 연산 기호 팔레트는 편집 영역 하단에 둔다.
- 점수와 시간은 짧은 숫자로 표시하고 설명 문구를 반복하지 않는다.
- 홈/설정/일시정지 같은 보조 액션은 44dp 원형 버튼으로 제한한다.
- 좌우 헤더 액션은 같은 `44×44dp` 크기로 맞춰 중앙 제목이 밀리지 않게 한다.
- Safe Area와 게임 헤더 사이에는 최소 `12dp`의 상단 여백을 둔다.
- 힌트의 남은 횟수는 버튼 폭을 늘리지 않고 작은 상태 배지로 표시한다.
- 기호 드롭 슬롯의 확장 피드백은 `70–90ms` 안에 끝내고 hover 상태가 바뀔 때만 다시 그린다.
- 보드 주변에는 장식용 일러스트, glow, gradient를 두지 않는다.
- 위험 상태가 아닐 때 빨강을 사용하지 않는다.

## 4. Arcade

- 지역/팩 카드는 공통 저채도 파스텔 토큰을 순환해 사용한다.
- `PLAY AT YOUR PACE` 같은 분위기용 작은 문구 없이 팩 이름과 행동만 보여준다.
- 활성 카드는 색면으로, 비활성 카드는 중립 Surface와 Hairline으로 구분한다.
- 진행률은 Ink 단색으로 표시한다.
- 화면의 가장 강한 CTA는 현재 레벨을 이어서 플레이하는 버튼 하나다.

## 5. Daily Challenge

- 날짜와 퍼즐 번호를 작은 Label 계층으로 표시한다.
- 날짜와 퍼즐 번호처럼 플레이에 실제로 필요한 메타데이터 외의 작은 설명은 넣지 않는다.
- 당일 도전 여부와 결과는 텍스트 및 미세한 배경톤으로 구분한다.
- 달력과 기록 카드에는 그림자를 사용하지 않는다.
- 공식 점수, 연습 점수, 랭킹 상태를 색상만으로 구분하지 말고 라벨을 함께 제공한다.

## 6. Ranking

- 순위 숫자, 닉네임, 점수 순으로 읽히게 한다.
- TOP 3에만 Lavender/Lime/Warm Gray 파스텔을 사용한다.
- 일반 행은 흰색 Surface와 Hairline을 사용한다.
- 장식용 메달, glow, 두꺼운 테두리는 사용하지 않는다.
- 사용자의 행은 굵기와 Soft Surface로 구분한다.

## 7. 결과 및 오류

- 결과 화면은 최종 점수와 다음 행동을 우선한다.
- 실패 배경 전체를 빨강으로 채우지 않는다.
- 오류 메시지는 간결하게 쓰고 재시도 액션을 바로 제공한다.
- Snackbar는 1dp 테두리와 평면 Surface를 사용하며 하드 섀도를 쓰지 않는다.

## 8. 검증 기준

- `flutter analyze`
- 전체 widget/unit test
- 홈: 844×390 및 667×375 overflow 검사
- 게임 화면: 작은 모바일 화면에서 HUD와 CTA 가림 여부 검사
- 텍스트 확대 시 핵심 점수와 액션이 잘리지 않는지 검사
