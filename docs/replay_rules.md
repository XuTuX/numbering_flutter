# NUMBERING 리플레이 기능 구현 규칙

이 문서는 NUMBERING 게임의 '리플레이 보기' 기능이 작동하는 핵심 규칙을 정의합니다. 본 규칙을 수정하거나 관련 로직을 변경할 때는 반드시 사용자의 허락을 받아야 합니다.

## 1. 데이터 기록 (Recording)
- **대상**: 사용자가 매칭에 성공한 모든 경로(`List<HexCoord>`)를 `recordedMoves` 리스트에 순서대로 기록합니다.
- **시간 정보**: 각 성공 경로에는 게임 시작 후 경과 시간(ms)을 함께 기록합니다. 서버는 이 값으로 콤보와 종료 시간을 다시 계산합니다.
- **조건**: 실제 플레이 중(`isReplaying == false`)일 때만 기록하며, 리플레이 중에는 기록하지 않습니다.
- **초기 상태**: 새로운 게임이나 다시 시작(`playAgain`) 시에는 기존 기록을 초기화하지만, 리플레이를 시작할 때는 기록을 보존합니다.

## 2. 결정론적 재현 (Deterministic Replay)
- **시드 활용**: 게임 시작 시 생성된 `initialSeed`를 보존합니다.
- **동일 환경**: 리플레이 시작 시 동일한 `initialSeed`를 사용하여 보드 레이아웃과 컬러 바Entry 순서를 정확히 복제합니다.
- **공통 난수**: 클라이언트와 PostgreSQL 검증 함수가 동일한 32비트 LCG 규칙을 사용합니다. 난수 규칙 변경은 리플레이 버전 변경과 서버 migration을 동반해야 합니다.

## 3. 리플레이 실행 (Playback)
- **상태 제어**: 리플레이 중에는 `isReplaying` 플래그를 `true`로 설정합니다.
- **입력 차단**: `isReplaying`이 `true`인 동안 사용자의 드래그 및 버튼 조작 입력을 차단하여 리플레이 방해를 방지합니다.
- **안내**: 화면 상단에 '리플레이 재생 중...' 배너를 표시하고 상태 메시지(`statusText`)를 통해 진행 상황을 알립니다.
- **지연 시간**: 각 수 사이에 일정 시간(약 600ms)의 지연을 두어 시각적 흐름을 보장합니다.

## 4. 시간 및 점수 처리 (Score & Time)
- **점수**: 리플레이 중에도 매칭에 따른 점수 증가와 애니메이션은 실제 게임과 동일하게 실행합니다.
- **타이머 정지**: 리플레이 중에는 시스템 타이머가 시간이 줄어들지 않도록 정지시키며, '시간 종료'로 인해 리플레이가 끊기지 않게 합니다.
- **시간 보너스 비활성화**: 리플레이 중에는 매칭 성공에 따른 시간 증가 보너스를 지급하지 않습니다.

## 5. 종료 및 전환 (Termination)
- **자동 종료**: 모든 기록된 수의 재생이 끝나면 안내 메시지를 표시한 뒤 일정 시간 후 자동으로 게임을 종료(`_endGame`) 처리합니다.
- **결과 화면**: 리플레이가 종료되면 '게임 오버' 오버레이가 다시 나타나며, 사용자가 다음 행동(홈, 랭킹, 다시 시작 등)을 선택할 수 있는 상태로 복귀합니다.

## 6. 서버 검증용 재생 데이터
- **형식**: 일반 기록은 경로별 경과 시간이 포함된 `HTR3:` 접두사를 사용합니다. 허니 러시가 발생한 일반 모드 기록은 발동 시각과 자유 드래그별 경로·시각을 추가한 `HTR4:`를 사용합니다.
- **용도**: 이 데이터는 서버가 공식 점수를 다시 계산하기 위한 내부 제출 형식이며 사용자 화면이나 공유 결과에는 노출하지 않습니다.
- **사용자 리플레이**: 사용자는 게임오버 화면의 `리플레이` 버튼으로 방금 끝난 자신의 플레이만 바로 확인합니다.
- **허니 러시 검증**: 서버는 `HTR4`에서 10칸 정상 매칭, 4초 입력 창, 인접 경로와 러시당 타일 1회 수집을 확인합니다.

## 7. 서버 권한 검증
- **신뢰 경계**: 서버는 클라이언트가 제출한 점수와 콤보를 신뢰하지 않습니다.
- **재계산**: 서버는 시드로 보드와 컬러 바를 재생성하고, 모든 경로의 좌표 범위, 중복, 육각 인접성, 컬러 바 일치, 타일/바 리필, 무지개 확률, 콤보 시간, 점수와 종료 상태를 순서대로 다시 계산합니다.
- **랭킹 반영**: 전체·주간·오늘 랭킹은 `submit_hexor_verified_run` 검증을 통과한 한 판만 반영합니다. 기존 점수 직접 제출 RPC는 NUMBERING 요청을 거절합니다.
- **모드 제한**: `HTR4`는 일반 모드 전체·주간 랭킹에만 허용하며 오늘의 퍼즐 공식 기록에는 허용하지 않습니다.
