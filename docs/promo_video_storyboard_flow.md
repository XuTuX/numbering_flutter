# NUMBERING 45초 세로형 홍보영상 스토리보드

## 1. 영상의 한 줄 방향

> 같은 색을 모으는 퍼즐이 아니다. 위에 보이는 **색의 순서**를 육각 보드에서 그대로 이어라.

- 목적: 처음 보는 사람이 10초 안에 게임 방법을 이해하고, 45초 안에 플레이 욕구를 느끼게 한다.
- 핵심 감정: `뭐지?` → `아, 이렇게 하는구나` → `길게 잇고 싶다` → `친구와 겨뤄 보고 싶다`
- 러닝타임: 45초
- 화면 비율: 9:16 세로
- 권장 출력: 1080 × 1920, 60fps
- 타깃: 짧게 즐길 퍼즐 게임을 찾는 10~30대 모바일 사용자
- 영상 방식: 실제 게임 녹화가 주인공. Flow는 오프닝, 확대, 전환, 배경 모션을 보조한다.
- 핵심 카피: `색의 흐름을 읽고, 한 번에 이어라.`

## 2. 전체 흐름

1. 가장 시원한 매치로 시선을 잡는다.
2. 컬러 바의 색 순서를 본다는 규칙을 알려 준다.
3. 같은 순서로 육각 타일을 드래그하는 모습을 보여 준다.
4. 긴 경로, 시간 보너스, 콤보의 보상을 보여 준다.
5. 오늘의 퍼즐과 랭킹으로 경쟁 욕구를 만든다.
6. 결과와 리플레이를 짧게 보여 주고 설치 문구로 끝낸다.

## 4. 먼저 녹화해야 할 실제 게임 클립

각 클립은 필요한 동작 앞뒤로 1초씩 여유를 두고 녹화한다. 편집할 때 HUD가 잘리지 않도록 화면 전체를 유지한다.

| 클립 | 반드시 담을 내용 | 사용 구간 | 합격 기준 |
| --- | --- | --- | --- |
| A | 홈 화면, 로고, 플레이 버튼 터치 | 0:02~0:05 | 알림이나 개인정보가 없고 로고가 선명함 |
| B | 컬러 바와 보드 전체가 보이는 게임 시작 | 0:05~0:09 | 점수·시간·컬러 바·보드가 모두 보임 |
| C | 정확한 순서의 3칸 매치 | 0:09~0:18 | 드래그 시작부터 제거와 리필까지 끊기지 않음 |
| D | 5~6칸 긴 매치 | 0:00~0:02, 0:18~0:23 | 경로가 겹치지 않고 손가락 이동이 쉽게 읽힘 |
| E | 3콤보 이상 | 0:23~0:27 | 4초 콤보 타이머와 콤보 수가 보임 |
| F | 허니 러시 또는 화려한 연속 매치 | 0:27~0:31 | 프레임 끊김 없이 가장 강한 손맛을 보여 줌 |
| G | 오늘의 퍼즐 카드와 입장 | 0:31~0:35 | `오늘의 퍼즐` 문구가 읽힘 |
| H | 랭킹 화면 | 0:35~0:39 | 오늘·주간·전체 중 두 가지 이상이 보임 |
| I | 게임 종료 결과와 리플레이 버튼 | 0:39~0:42 | 점수, 콤보, 버튼이 선명함 |
| J | NUMBERING 로고 PNG | 0:02~0:05, 0:42~0:45 | 투명 배경 또는 깨끗하게 분리된 원본 사용 |

### 녹화 권장 순서

1. 알림 방해 금지, 화면 밝기, 한국어 설정을 확인한다.
2. 홈 화면 A와 오늘의 퍼즐 G를 먼저 녹화한다.
3. 일반 플레이 한 판을 처음부터 끝까지 녹화해 B, C, D, E 후보를 만든다.
4. 긴 매치와 콤보가 약하면 일반 플레이를 한 번 더 녹화한다.
5. 허니 러시 F는 별도 테이크로 확보한다. 실패하면 화려한 매치 두 개로 대체한다.
6. 게임 종료 후 결과 I, 랭킹 H, 리플레이 시작 장면을 이어서 녹화한다.
7. 모든 원본을 확인한 뒤 Flow에는 필요한 구간만 잘라 넣는다.

## 5. Flow에 넣을 전체 지시문

아래 지시문과 함께 실제 게임 클립, `assets/icons/logo.png`, 필요하면 `assets/icons/graphic image.png`를 참조 자료로 제공한다. Flow가 한 번에 45초 전체를 안정적으로 만들지 못하면 다음 절의 장면별 프롬프트로 나눠 생성한다.

```text
Create a 45-second vertical 9:16 mobile game promotional video for “NUMBERING.”
Use the uploaded real gameplay recordings as the primary visual source. The core idea is:
“Do not match identical colors. Read the color sequence at the top and connect adjacent hex tiles in exactly the same order.”

Tone: energetic, clever, satisfying, colorful, modern casual puzzle game.
Pacing: fast hook in the first 2 seconds, clear tutorial from 5 to 18 seconds, escalating rewards and combos from 18 to 31 seconds, competition and replay features from 31 to 42 seconds, clean logo CTA from 42 to 45 seconds.

Preserve the original gameplay UI exactly. Do not redesign, regenerate, replace, recolor, or distort the board, hex tiles, color stream, Korean text, score, time, combo counter, rankings, or buttons. Do not invent gameplay. Use only real successful moves from the provided recordings. Add only simple editorial overlays: crop-safe zooms, thin highlight outlines, numbered markers, short motion transitions, and subtle hexagonal background graphics.

Format: 1080x1920, 60 fps, full-screen portrait gameplay. Keep the score, timer, top color stream, and board visible whenever they are needed to explain the rules. Use direct cuts on sound beats. Use a brief slow-down only while demonstrating a long drag, then return to normal speed at the success moment.

Visual sequence:
1) Open on the most satisfying 5-6 tile successful match.
2) Show the NUMBERING home screen and tap Play.
3) Zoom toward the top color stream and highlight one continuous 3-color segment.
4) Show a finger dragging through three adjacent hex tiles in that exact order.
5) Show the real clear, refill, score increase, and time reward.
6) Show a longer 5-6 tile match.
7) Show fast consecutive matches and the real combo timer.
8) Show Honey Rush only if it exists in the uploaded real gameplay; otherwise use two strong real match clips.
9) Show Today’s Puzzle and ranking screens.
10) Show the Game Over result and tap Replay.
11) End on the provided NUMBERING logo with a clean App Store call to action.

Do not generate Korean lettering inside the imagery. Reserve clean lower-third and center-safe areas for captions that will be added in editing. Do not add a fake smartphone frame, fake hands, characters, bees, honey, coins, or gameplay elements that are not present in the supplied assets.
```

## 6. Flow 장면별 프롬프트

Flow가 실제 게임 화면을 임의로 바꾸는 것을 줄이기 위해 각 프롬프트 맨 앞에 아래 공통 문장을 붙인다.

```text
Use the uploaded gameplay clip exactly as recorded. Preserve every UI element, tile color, Korean label, number, and gameplay action. Do not generate or replace gameplay content.
```

### 장면 1 — 오프닝 훅, 0:00~0:05

```text
Create a five-second vertical opening. Begin immediately on the supplied successful long-match gameplay clip. Use a subtle 4% push-in while the drag completes, then add a short impact at the real clear moment. Hard cut to the supplied NUMBERING home screen and logo. End as the real Play button is tapped. Energetic and satisfying, no extra objects, no fake interface, no generated text.
```

### 장면 2 — 게임 방법, 0:05~0:18

```text
Create a thirteen-second vertical tutorial sequence from the supplied gameplay. First show the full HUD. Smoothly crop toward the top color stream and place a thin, non-obstructive outline around one real continuous three-color segment. Return to the full board and show the real finger drag through three adjacent hex tiles in exactly the same order. Add small 1, 2, 3 editorial markers beside the selected tiles without covering them. Let the original clear and refill animation finish, then briefly emphasize the real score and time increase. Keep the instruction visually understandable even with audio muted.
```

### 장면 3 — 긴 경로와 콤보, 0:18~0:31

```text
Create a thirteen-second vertical escalation montage using only the supplied real successful gameplay. Show one 5-6 tile path at 90% speed so the route is readable, returning to normal speed at the success moment. Then cut on beats between consecutive successful matches while keeping the real combo timer visible. If a genuine Honey Rush clip is supplied, use it as the final four-second climax. If not, finish with the two strongest real clear-and-refill moments. Use restrained camera punch-ins only; never change gameplay outcomes or numbers.
```

### 장면 4 — 경쟁과 기록, 0:31~0:42

```text
Create an eleven-second vertical feature montage from the supplied app recordings. Show the Today’s Puzzle card, the real daily game entry, a short real ranking-list scroll, the Game Over result card, and a tap on Replay. Use clean direct cuts and subtle hexagon-shaped wipes. Keep all personal information out of frame or use only approved test-account footage. Do not modify names, ranks, scores, or buttons.
```

### 장면 5 — 엔드 카드, 0:42~0:45

```text
Create a clean three-second vertical end card using the supplied original NUMBERING logo. Use a warm off-white background with very subtle low-opacity hexagon shapes drifting slowly. Keep the center and lower third clear for a Korean tagline and App Store call to action added later. No generated lettering, no fake store badge, no mascot, no additional icons. Finish with the logo fully still for the final one second.
```

## 9. 편집 체크리스트

- 첫 2초에 로고가 아니라 실제 성공 장면이 나오는가?
- 10초 안에 `상단 색 순서 → 같은 순서로 드래그`가 이해되는가?
- 잘못된 순서, 비인접 타일, 3칸 미만 경로를 성공처럼 보이게 하지 않았는가?
- 점수, 시간, 콤보 숫자는 실제 게임 녹화와 일치하는가?
- 모든 자막이 모바일 화면에서 1초 이상 읽히는가?
- 자막이 컬러 바, 점수, 시간, 콤보, 손가락 경로를 가리지 않는가?
- 허니 러시는 실제 10칸 매치로 발동한 원본 장면만 사용했는가?
- 랭킹 화면에 동의받지 않은 닉네임이나 개인정보가 없는가?
- 효과음이 성공 시점과 정확히 맞는가?
- 마지막 로고와 CTA가 최소 1초 동안 정지해 있는가?
