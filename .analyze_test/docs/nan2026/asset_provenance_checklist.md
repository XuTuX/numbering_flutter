# 외부 에셋 및 라이선스 확인표

NAN 2026은 외부 이미지·사운드의 출처와 라이선스를 AI 활용 기술 문서에 반드시
기재하도록 요구한다. 아래 `확인 필요` 항목을 사실에 맞게 채운 뒤 PDF를 재생성한다.

| 파일/범위 | 현재 용도 | 출처/권리 상태 | 제출 전 증빙 |
| --- | --- | --- | --- |
| `assets/icons/logo.png` | 앱 아이콘/브랜드 | 확인 필요 | 직접 제작 원본, 생성 도구 내역, 또는 라이선스 |
| `assets/icons/graphic image.png` | 프로모션 그래픽 | 확인 필요 | 제작 원본 또는 생성 도구/프롬프트 |
| `assets/icons/google_logo.png` | Google 로그인 | Google 브랜드 자산 추정, 확인 필요 | 공식 브랜드 가이드 URL과 사용 조건 확인 |
| `assets/bgm/home_background.mp3` | 홈 BGM | 확인 필요 | 제작자, 구매처/라이선스, 또는 생성 도구 내역 |
| `assets/bgm/background_bgm.mp3` | 게임 BGM | 확인 필요 | 제작자, 구매처/라이선스, 또는 생성 도구 내역 |
| `assets/bgm/clear_bgm.mp3` | 결과/클리어 사운드 | 확인 필요 | 제작자, 구매처/라이선스, 또는 생성 도구 내역 |
| `assets/bgm/click.mp3` | UI 클릭 사운드 | 확인 필요 | 제작자, 구매처/라이선스, 또는 생성 도구 내역 |
| `pubspec.yaml` 직접 의존성 | 앱 기능 | 각 오픈소스 프로젝트 LICENSE 적용 | 버전/홈페이지/라이선스 목록을 PDF 최종본에 유지 |

## 권리 상태 작성 예시

- 직접 제작: `제작자 나찬혁, 2026-04-XX, 원본 프로젝트 보관`
- AI 생성: `도구명/모델, 생성일, 대표 프롬프트, 상업 이용 약관 URL`
- 라이선스 구매: `서비스명, 에셋 URL, 라이선스명, 구매 증빙 보관`
- 무료 에셋: `제작자, 원문 URL, 라이선스명, 저작자 표시 문구`

추정으로 채우지 말고 실제 제작·구매 기록에 근거해 작성한다.
