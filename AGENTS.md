# Project UI rules

- Keep all visible UI copy minimal and functional.
- Never add decorative, congratulatory, explanatory, secondary small subtitles, or redundant status phrases such as “new high score”, rank summaries, “play next”, “open”, “locked”, “complete”, “01 - 20”, or pack/mode subtitles.
- Never display secondary small text labels, level range indicators (e.g. "01 - 20"), mode duration subtitles (e.g. "3 MIN"), or small descriptive captions below main titles.
- Prefer numbers, icons, progress indicators, and visual state over explanatory labels.
- Add visible text only when it is essential to understand or operate the interface, and use the shortest clear wording.
- Rankings must use verified official daily-puzzle results only.
- Arcade play stores level progress locally and must never submit scores to, or affect, any ranking.
- Generated arcade levels must have distinct normalized equation shapes and must not repeat the same shape in adjacent levels.
